## 하나의 예외 타입만으로도 충분합니다 (171p)

단 하나의 예외 객체만 가진다는 말이 무슨 말인지 모르겠음!!
어차피 계속 예외를 체이닝하고 단 한번만 복구한다면, 예외 상황에 맞는 타입을 여러개 써도 상관없지 않나?
예외 타입을 하나만 사용한다는 것의 이점을 모르겠음.. 아니 그냥 이해를 잘 못했다. 

**-> 예외 타입에 따라 분기 처리하는 것이 아니고, 어떤 예외 타입인지 상관없이 다시 예외를 체이닝한다.   
즉, 굳이 여러 예외 타입을 나눌 필요가 없다!**


<br>

## RAII를 사용하세요 (178p)

RAII의 정의가 정확하게 이해가 안 간다. 
책에서 RAII를 왜 사용하라고 하는 건지, RAII를 사용했을 때의 이점은 무엇인지 파악하지 못했다.

**-> `객체를 초기화할 때 리소스를 확보하고, 객체가 더이상 사용되지 않을 때 리소스를 해제하는 것.`   
객체를 더이상 사용하지 않는다면 명시적으로 종료하자.   
코틀린이나 자바에서는 `AutoCloseable`을 사용할 수 있다.**
